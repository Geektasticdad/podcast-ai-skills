# Standalone Episode Narrative Builder

This skill takes a completed standalone episode outline and writes the full spoken narrative for every section. The output is a formatted Word docx generated by running generate-docx.py against a structured JSON file.

This is a standalone episode narrative. There is no series recap section.

This skill asks two questions before producing the narrative.

**Requires:** Load the Research Foundation skill (`foundation/SKILL.md`) alongside this skill before you begin.

**Requires python-docx.** If not installed:
```
pip install python-docx
```

**Requires Episode TBD.docx** in the `assets/` subdirectory of this skill. This file provides the branded style template for the exported document.

## Inputs

Ask the user the following questions in sequence. Wait for each answer before asking the next.

**Question 1 (optional draft file):**
"Do you have an episode draft JSON file from the outline stage? If yes, provide the file path. If no, press enter."

If a valid file path is provided, read and parse the draft JSON:
- If `outline.outline_text` is set, skip Question 2. Pre-fill the episode title from `outline.selected_title` if set, or from `ideas.selected_idea.title` as a fallback. Confirm: "Using outline for: [title]. Continuing."
- If `narrative.episode_number` is already set, skip Question 3. Confirm: "Using episode number: [value]. Continuing."
- If `meta.stage` is already "complete", ask: "This draft already has a completed narrative. Do you want to regenerate it?"

**Question 2:**
"Paste the full output from your `podcast-episode-outline` for the episode you want to write."

**Question 3:**
"What is the episode number? If not yet assigned, type TBD."

Once you have all needed answers, produce the full narrative. Do not ask any further questions.

## Output Process

After writing the narrative, do the following:

1. Write the narrative content to a JSON file named `podcast-episode-temp.json` in the current working directory, following the JSON schema below.
2. Locate `scripts/generate-docx.py` in this skill's directory.
3. Run the script: `python scripts/generate-docx.py podcast-episode-temp.json`
4. Delete `podcast-episode-temp.json` after the docx generates successfully.
5. If a draft file was loaded in Question 1:
   - Update the draft JSON: populate `narrative.episode_number`, `narrative.title`, `narrative.episode`, `narrative.sections`, `narrative.references`, `narrative.output` from the completed narrative payload; set `meta.stage` = "complete" and `meta.last_updated_at` to the current timestamp.
   - Write the updated draft file back to the same path.
   - Report: "Draft updated: [filename] (stage: complete)."
   If no draft file was loaded, ask: "Would you like to save the completed narrative to a draft file for archiving? Type a filename or 'no'."
6. Report the docx filename and the directory it was saved to.

The docx saves to the current working directory as `Episode_[Episode Number].docx`.

## JSON Schema

Generate the narrative following the structure defined in `references/episode-template.json` in this skill's directory. Every paragraph slot must contain fully written narrative prose. Do not include any label prefixes or placeholder text in the final output.

For the opening sign-on, use the exact text from the `SHOW_SIGNON` context variable set in the foundation. For the closing sign-off, use the exact text from the `SHOW_SIGNOFF` context variable.

Scale the number of Topic Discussion subsections to match what the outline provides. Add or remove subsections as needed. Do not leave placeholder headings or paragraph text in the final JSON — every field must contain fully written content.
